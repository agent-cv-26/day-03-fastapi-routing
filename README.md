# 🚀 اليوم 03: إعداد FastAPI والـ Routing المتقدم

هذا المستودع هو جزء من **خارطة الطريق التقنية (120 يوماً)** للوصول إلى مستوى مهندس ذكاء اصطناعي خبير. تركيزنا اليوم كان على بناء هيكل مشروع احترافي وفهم نظام التحقق من جودة البيانات.

## 🛠️ التنفيذ التقني
- **البيئة الأسبوعية:** إنشاء بيئة Conda مخصصة `week-01-fastapi` لعزل المكتبات وضمان استقرار النظام.
- **خادم ASGI:** استخدام `uvicorn` كخادم لإدارة دورات الاستجابة والطلب غير المتزامنة.
- **مسارات المسار (Path Parameters):** بناء مسارات ديناميكية للتعامل مع معرفات العناصر `item_id` ومعرفات النماذج `model_id`.
- **التحقق من البيانات:** الاستفادة من مكتبة **Pydantic** للفحص التلقائي للأنواع (تم إثبات ذلك عبر اختبار خطأ HTTP 422).

## 🧪 اختبار الـ API
1. تشغيل السيرفر: `uvicorn app.main:app --reload`
2. الدخول إلى واجهة Swagger UI: `http://127.0.0.1:8000/docs`
3. اختبار البيانات الخاطئة: إدخال "نص" في حقل يتوقع "رقماً" لرؤية استجابة الخطأ **422 Unprocessable Entity**.
